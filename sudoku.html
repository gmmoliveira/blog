Solving a sudoku as a Mixed Integer Programming problem

Introduction

A classical N-sudoku problem refers to a sudoku of size N, e. g., a sudoku of size N=9 would be formed by a 2D grid of 9 by 9 elements, 81 cells in total. The grid would also be subdivided in 9 boxes of dimensions 3 by 3 cells each, where the number 3 is obtained by taking the square root of N: √9=3. Further descriptions of the particular 9-sudoku game may be found here, however, this work is concerned about sudoku games of an arbitrary size N, where valid sizes are non-negative integers squared, i. e., N ∈ ℕ2. This article will briefly explain how to model such problem as a MIP (Mixed Integer Programming), proposing a practical python implementation, the source-code is available at my github

If you'd like to understand further, yet in a introductory level, how the general version of Linear Programming and Mixed Integer Programming works, in addition to seeing how I have proposed to facilitate it through the call of a single python function, you should refer to this article. The framework used in the background is Google OR-Tools.

Modeling the sudoku as a MIP problem

All MIP variables have the format Xijk which means wether cell at position (i, j) in the sudoku table should be assigned number k + 1 or not. They may assume only integer binary values Xijk ∈ {0, 1} ∀ i, j, k ∈ {0, N - 1} and i, j, k ∈ ℕ, where the sudoku table is a N by N grid. The number of variables in the MIP model is denoted by n, it's different from the capital N that defines the dimension of the sudoku. The number of variables in the model is given by n = N3 since for each sudoku cell we have a one-hot-encoded (binary) vector representing which number should be assigned at that cell. Before diving into the relevant coding, let's assume we have this initial setup: 

import numpy as np


indexing_encoder = lambda i, j, k: (i * n + j) * n + k
A = np.zeros(shape=(4 * (n ** 2) + nonempty_values, n ** 3), dtype=int)
b = np.ones(shape=A.shape[0], dtype=int)
C = np.ones(shape=A.shape[1], dtype=int)
constraint = 0
indexing_encoder is a convenience lambda function to standardize the indexing of variables Xijk in matrix A, by it's turn A is a matrix of constraints, b is a vector representing both the lower and upper bounds of constraints simultaneously (each equality constraint is defined in practice as 2 tight inequality constraints, note that the sudoku game requires equality constraints which might not be the case for different problems), C is the objective function, constraint is an auxiliary variable for continuously indexing the constraints in matrix A throughout the creation of each set of constraints. There is also a variable called sudoku which is a N by N numpy array representing the initial state of the sudoku, where cells filled in with a 0 are considered empty, but cells filled with any other number will be assumed to contain, indeed, that number. A is assumed to be initialized with zeros so we may fill in 1s where required, while b and C are assumed to be initialized with ones (we don't have to change them throught the process like we do for matrix A). The constraint variable is initialized with a 0. Further, the MIP constraints are created as follows:

Rows constraints


while constraint < (n ** 2):
	for i in range(0, n):
		for k in range(0, n):
			for j in range(0, n):
				A.itemset(constraint, indexing_encoder(i, j, k), 1)
			constraint += 1

Columns constraints



while constraint < 2 * (n ** 2):
	for j in range(0, n):
		for k in range(0, n):
			for i in range(0, n):
				A.itemset(constraint, indexing_encoder(i, j, k), 1)
			constraint += 1

Boxes constraints

The lowercase n and m refers to the boxes which may be indexed in each dimension of rows and columns of the sudoku table, furthermore, in this work, the sudoku version implemented always defines exactly N boxes, evenly distributed without overlapping across the sudoku table, this means n=√N and m=√N :

The above image might lead to confusion since the indexes under the sums are too close to each other, however, the first sum is indexed by variable i, starting at i = (I - 1)m + 1, while the second one is index by variable j, starting at j = (J - 1)n + 1. 

sqrtn = int(n ** (1 / 2))  # n always have an integer square root
for boxcol in range(0, sqrtn):
	for boxrow in range(0, sqrtn):
		for k in range(0, n):
			for i in range(boxrow * sqrtn, (boxrow + 1) * sqrtn):
				for j in range(boxcol * sqrtn, (boxcol + 1) * sqrtn):
					A.itemset(constraint, indexing_encoder(i, j, k), 1)
			constraint += 1

Uniqueness constraints

These constraints determines that only a single number may be assigned for each sudoku table cell. 

while constraint < 3 * (n ** 2):
	for i in range(0, n):
		for j in range(0, n):
			for k in range(0, n):
				A.itemset(constraint, indexing_encoder(i, j, k), 1)
			constraint += 1

Prespecified cell values constraints

This type of constraints aims at making sure that the sudoku obeys any cells which are already filled with values at the start of the game. 

for i in range(0, n):
	for j in range(0, n):
		cell_value = sudoku.item(i, j)
		if cell_value != 0:
			A.itemset(constraint, indexing_encoder(i, j, cell_value - 1), 1)
			constraint += 1

Software documentation

Implements a class Sudoku, with the following methods:
  • Sudoku.__init__(self, sudoku=None, n=9) : Builds an object which represents a n-sudoku.
    • sudoku: a sudoku state, as a 2D numpy array, to be used as the initial state for this object. Defaults to None, where a random sudoku initial state will get created, it's going to start with a few cells already filled with values, like a real sudoku game would;
    • n: size of the n-sudoku in a single dimension, a sudoku of dimensions n by n gets created;
    Raises an Exception if √n isn't an integer;
  • Sudoku.linear_programming_model(self) : generates a valid linear programming (LP) model for the sudoku represented by this object;
  • Sudoku.linear_programming_solve(self) : solves the generated linear programming model;
  • Sudoku.random(self) : generates a new n by n sudoku with a few random entries already filled (as would be a regular sudoku challenge be), the generated states might be biassed towards alternate optima sudoku scenarios;
  • Sudoku.get_puzzle_state(self): returns a 2D NumPy array representing the sudoku in it's current state;
  • Sudoku._lpsolution2sudoku(self): decodes the LP solution array X* into a valid sudoku state;

Executing the src/sudoku.py file outputs two examples, where the matrices represents the sudoku tables. The first table is a initial game state where number 0 represents that the corresponding table cell is empty, the second one refers to the same table after being solved, this goes for two examples: 3-sudoku and 4-sudoku. Conclusion

Since all constraints in the MIP model are of equality type without slack variables, rather than inequality, maximizing or minimizing the objective function, defined as an array of 1s, yields the exact same optimum objective value. It doesn't necessarily lead to the exact same optimum solution vector X*, since a single sudoku might be loosely constrained, therefore, it would accept multiple different solution vectors. If the sudoku is tightly constrained, only a single solution vector X* will exist.
